# OOP-Private, Public
- public/private 메소드
- 공개할 수 있는 정보 : Public
- 공개하면 안되는 정보: Private

```Ruby
class Person
    # 공개되도 괜찮은 정보
    def initialize(name, age)
        @name = name
        @age = age
    end

    def run
        # 맨 마지막 줄이 return이 되는데 ruby에서는 따로 return 명령어 필요가 없다. 
        "#{@age}살의 #{@name}가 달립니다"
    end

    # 공개되면 안되는 정보
    def your_password
        @password = "123456789"
    end
end

dodo = Person.new("도도", 30)
puts dodo.run 
puts dodo.your_password 
# 비밀번호도 마음만먹으면 출력되는 문제가 발생
# 아무나 your_password에 접근하지 못하도록 만들어야 한다.
```

```Ruby
class Person
    def initialize(name, age)
        @name = name
        @age = age
    end

    public 
    def run
        "#{@age} 살 #{@name}가 달립니다"
    end

    private
    def your_password
        @password = "123456789"
    end
end

p = Person.new("도도", 30)
puts p.run
puts p.your_password #=> 접근 불가 NoMethod Error (Private변수이기때문이다.)
```

## Object 밖에서 접근할 수 없는 private method
-  `object 내부`에서는 접근 가능


```Ruby
class Person
    def initialize(name, age)
        @name = name
        @age = age
    end

    public
    def run
        "#{@age} 살의 #{@name}가 달립니다."
    end

    def check_password(password)
        # 오브젝트안에서는 private 변수 마음대로 접근 가능
        if password == your_password
            puts "계좌에 접근하실 수 있습니다."
        else 
            puts "비밀번호가 틀렸습니다."
        end
    end

    private 
    def your_password
        @password = "123456"
    end
end

p = Person.new("dodo", 20)
puts p.run
puts p.check_password("12345") #-> 비밀번호가 틀렸습니다. 출력
puts p.check_password("123456") #-> 계좌에 접근하실 수 있습니다. 출력
```
